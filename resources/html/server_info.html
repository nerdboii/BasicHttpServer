Simple HTTP Server Overview

Welcome to the documentation page for our Simple HTTP Server implementation. This server provides lightweight HTTP functionalities designed for high concurrency and performance, supporting both static content serving and basic API responses.

Features

  • HTTP/1.1 protocol support
  • Supports GET and HEAD methods
  • Handles up to 10,000 concurrent client connections
  • Processes up to 100,000 requests per second
  • Minimal and efficient C++ implementation
  • Configurable document root for static file serving
  • Simple logging of request events

HTTP Methods Supported

Method Description
GET Requests a representation of the specified resource. Returns the resource content in the response body.
HEAD Same as GET, but without the response body. Returns only headers.

Sample HTTP Request

GET /index.html HTTP/1.1
Host: localhost:8080
User-Agent: curl/8.0.1
Accept: */*

Sample HTTP Response

HTTP/1.1 200 OK
Content-Type: text/html
Content-Length: 4523

<!DOCTYPE html>
<html>
  <head><title>Welcome</title></head>
  <body>
    <h1>Welcome to the HTTP Server</h1>
  </body>
</html>

Performance Benchmarks

Metric Value
Maximum Concurrent Clients 10,000
Peak Request Handling Rate 100,000 requests/sec

Server Root Directory

All resources are served from the following directory:

/var/www/html/

Ensure all static content such as .html, .css, .js, images, and other files are placed within this folder or its subdirectories.

Error Responses

  • 404 Not Found: Returned when a requested resource does not exist on the server.
  • 501 Not Implemented: Returned when an unsupported HTTP method is requested.

Example 404 Response

HTTP/1.1 404 Not Found
Content-Type: text/plain
Content-Length: 13

404 Not Found

About This Server

This HTTP server was built in C++17 using the std::filesystem and epoll APIs for efficient file handling and network event management. It aims to demonstrate fundamental server architecture principles while maintaining simplicity and high performance.
